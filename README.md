## The Origin Story

I started in a laboratory.
Not a code lab, a medical laboratory. Analyzing blood samples. Running diagnostics. Following protocols where a single error could mean a misdiagnosis. That environment taught me something that most developers learn the hard way: *precision matters, but documentation saves lives.*

When I pivoted to digital journalism, I discovered another truth: *complexity is only a communication problem.* I spent years translating dense policy documents, medical research, and tech developments into stories that 570+ articles later, still resonate with readers.

Then I found **Google Apps Script**.
And realized I'd been preparing for this my entire career.

## The Philosophy

> "Every system I ship functions efficiently and is documented impeccably."

Most automation breaks in the dark, at 2 AM, when the person who built it has left the company, when the documentation is a cryptic comment saying `// TODO: fix this later`.

I build systems that **outlast their builder.**
Every script. Every workflow. Every dashboard. Designed for the next engineer, the next stakeholder, the next 3 AM emergency. Because *code that can't be maintained is technical debt wearing automation's clothes and what was not documented, was not done.*
